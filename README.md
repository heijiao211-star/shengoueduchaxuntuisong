# 纳指100场外申购额度监控

每天工作日北京时间 09:10 自动抓取配置基金的公开交易状态，计算每只基金的单日上限与建议申购金额，并通过 PushPlus 推送到微信。

## 日报字段

| 字段 | 含义 |
|---|---|
| 基金代码 / 名称 / 公司 | 便于快速定位产品 |
| 定投标记 | `已定投` 或 `未定投`，由配置文件维护 |
| 计划申购 | 自己计划每天投的金额，默认 ¥100 |
| 今日状态 | 开放申购、限大额申购、暂停申购或获取失败 |
| 单日上限 | 公开交易页面披露的当日累计购买上限 |
| 今日建议申购 | `min(计划申购, 单日上限)`；暂停或异常时为 ¥0 |
| 定投状态 | 页面可识别到的定投起投信息 |

微信内容会以卡片式 HTML 表格展示；仓库同时保留最新报告和逐日历史记录。

## 首次配置

1. 建议将仓库设为 **私有**，避免公开你的定投标记和计划金额。
2. 在 GitHub 仓库打开 `Settings → Secrets and variables → Actions`，新建 Secret：`PUSHPLUS_TOKEN`。
3. 在 PushPlus 公众号/后台确认微信已绑定并开启一对一消息接收。
4. 打开 `Actions → 纳指100申购额度日报 → Run workflow`，手动运行一次验证推送。

> 不要把 PushPlus token 写入代码、Issues、README 或提交记录。

## 修改基金或申购金额

编辑 [`config/funds.json`](config/funds.json)：

```json
{"code": "000834", "invested": true, "planned_amount": 100}
```

`invested` 控制“已定投”标记；`planned_amount` 是你准备投的金额。若实际下单金额不同，只改这一项即可。

## 数据边界

监控读取的是公开基金档案中展示的交易状态与单日购买上限。它不能登录或读取支付宝账户，因此支付宝最终下单是否成功、渠道临时限额和当日额度是否耗尽，仍须以支付宝页面为准。
